API 概要

GIGA SCHEMAで利用出来るAPIについて解説します。このAPIではGIGA SCHEMAに対して以下の操作を行うことができます。なお、このAPIではschemaの作成や設定を行うことができません。GIGA SCHEMAのWebサイトでschemaを作成後、ご利用ください。

  • schema操作
    • schemaへのdataの記録
    • schema情報の取得
    • schema内のdataをランダムで取得する
  • data操作
    • data情報の取得(JSON)
    • data情報の取得(raw)

APIに対してはHTTPによりリクエストを発行します。特に明記しない限りレスポンスはJSON形式になります。

このAPIは、XMLHttpRequest Level 2 に対応しています。リクエストを許可するドメインはGIGA SCHEMA Webサイト内のschema設定ページから指定できます。デフォルトではすべてのWebサイトからのリクエストを許可しています。


schema操作

schemaへのdataの記録

特定のschemaにdataを記録することができます。記録時にgroupを指定することでdataを取得する際に絞り込みを行うことができます。また、value を複数渡すことで同時に複数のdataを記録できます。

dataが記録されると、自動的にvalueの種類が判定されます。

リクエスト
 
POST /{user_name}/{schema_name}.json
パラメータ値
valuedataとして記録する値(複数可)
groupdataを取得する際に絞り込みに利用可能な任意の文字列
レスポンス
 
{ 
    "name" : "sample",  // スキーマの名前 
    "data" : [ 
        "group"      : "",
        "value"      : "これはサンプルです",
        "item_type"  : "text",               // 自動的に識別されたvalueの種類 
        "created_on" : 1293269403,           // データ作成日時 
        "key"        : "agpnaWdhc2NoZW"      // データに割り当てられたユニークな文字列 
    ] 
}

schema情報の取得

特定のschemaの情報および、schemaに記録されているdataを取得します。ページングやgroupパラメータによる絞り込みに対応しています。

リクエスト
 
GET /{user_name}/{schema_name}.json
パラメータ値
page取得するdataのページ数
group取得するdataの絞り込みに用いるgroup
api_keyAPIキーが指定されたschemaにアクセスする際に必要
レスポンス
 
{ 
    "name" : "sample",  // スキーマの名前 
    "page" : 1,
    "has_next" : 1,     // 次のページが存在するかを示す真偽値 
    "has_prev" : 0,     // 前のページが存在するかを示す真偽値 
    "data" : [ 
        { 
           "group"      : "",
            "value"      : "これはサンプルです",
            "item_type"  : "text",               // 自動的に識別されたvalueの種類 
            "created_on" : 1293269403,           // データ作成日時 
            "key"        : "agpnaWdhc2NoZW"      // データに割り当てられたユニークな文字列 
        },
        { 
            "item_type"  : "text",
            "group"      : "",
            "value"      : "サンプル2",
            "created_on" : 1293269403,
            "key"        : "agpnaWdhc2NoZW" 
        },
        // .. 
    ],
}

schema内のdataをランダムで取得する

特定のschemaに含まれるdataをランダムに取得することができます。

リクエスト
 
GET /{user_name}/{schema_name}/random.json
レスポンス
 
{ 
    "group"      : "",
    "value"      : "スキーマ内のどのデータが取得できるかはわかりません",
    "item_type"  : "text",               // 自動的に識別されたvalueの種類 
    "created_on" : 1293269403,           // データ作成日時 
    "key"        : "agpnaWdhc2NoZW"      // データに割り当てられたユニークな文字列 
}

data操作

data情報の取得(JSON)

指定したキーのdataを取得できます。

リクエスト
 
GET /{user_name}/{schema_name}/{data_key}.json
レスポンス
 
{ 
    "group"      : "",
    "value"      : "サンプル",
    "item_type"  : "text",               // 自動的に識別されたvalueの種類 
    "created_on" : 1293269403,           // データ作成日時 
    "key"        : "agpnaWdhc2NoZW"      // データに割り当てられたユニークな文字列 
}

data情報の取得(raw)

指定したキーのdataを取得できます。レスポンスにははJSON形式ではなくvalueがそのままのかたちで含まれます。

リクエスト
 
GET /{user_name}/{schema_name}/{data_key}.value
